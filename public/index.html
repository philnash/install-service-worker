Service worker installation methods

This small app shows the different ways you can install a service worker. When service workers were first released you needed to use the JavaScript API, but Chrome 54 shipped two new methods, installation via the HTML <link> element or by an HTTP Link header.

See below for the various methods to install a service worker that is available at /sw.js with a scope of "/".

JavaScript installation

The original and best for cross browser compatibility, responding to errors and doing advanced work with a service worker registration, like subscribing to push notifications:

navigator.serviceWorker.register("/sw.js", { scope: "/" })

See running example.

<link> element registration

A simple, declarative method of registering a service worker in the <head> of a document:

<link rel="serviceworker" href="/sw.js" scope="/">

See running example.

HTTP Header

No extra markup and an installed service worker with just an extra HTTP header:

Link: </sw.js>; rel="serviceworker"; scope="/"

See running example.
